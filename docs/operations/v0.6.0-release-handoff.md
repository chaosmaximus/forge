### 7.2 — Tag and push v0.6.0

The CI release workflow (`.github/workflows/release.yml`) triggers
on `v*` tag pushes. It builds **3 targets** (aarch64-apple-darwin,
x86_64-apple-darwin, x86_64-unknown-linux-gnu) and uploads tarballs
as a draft release. The Linux tarball you'll get from CI should
match the locally-staged one byte-for-byte (assuming GHA runner has
the same toolchain — verify if you want via sha256).

```bash
cd /mnt/colab-disk/DurgaSaiK/forge/forge
git tag -a v0.6.0 -m "Forge v0.6.0 — first tagged release"
git push origin v0.6.0
```

CI will produce a draft release within ~10 min once it builds all
three targets.

### 7.5 — Update Formula/forge.rb sha256 placeholders

After `gh release create` lands, the release artifacts have stable
download URLs. Compute sha256 for each tarball and replace the
`PLACEHOLDER` lines in `Formula/forge.rb`:

```bash
URL_LINUX=https://github.com/chaosmaximus/forge/releases/download/v0.6.0/forge-0.6.0-x86_64-unknown-linux-gnu.tar.gz
SHA_LINUX=$(curl -fsSL "$URL_LINUX" | sha256sum | cut -d' ' -f1)
echo "Linux: $SHA_LINUX"
# Repeat for the two macOS targets when Macs build them.

# Commit Formula/forge.rb update:
git checkout -b chore/forge-formula-v0.6.0-sha
# (edit Formula/forge.rb — replace each PLACEHOLDER with the matching sha256)
git add Formula/forge.rb
git commit -m "chore: forge.rb sha256 for v0.6.0 release tarballs"
git push origin chore/forge-formula-v0.6.0-sha
gh pr create --fill --base master
```
